Introduction

Добро пожаловать в AsmX, мощный язык программирования для низкоуровневого системного программирования. AsmX позволяет вам писать высокоэффективный и оптимизированный код, который может выполняться в самых разных системах, от встроенных устройств до больших серверов. Прежде чем мы углубимся в детали AsmX, давайте сначала рассмотрим некоторые основные концепции, которые вам необходимо понять. Эти концепции являются неотъемлемой частью низкоуровневого программирования и обеспечат прочную основу для вашего путешествия с AsmX. 

        basic program in the AsmX programming language

        
        # Copyright (c) 2023
        # GitHub (E5war5IT)
        # ./examples/hello.asmX
        @set hello String "Hello AsmX!"; 
        
        @memory hello  0x01  # write to memory
        @address 0x01 txt # define address of the memory
        
        @Route  txt 0x01 # route
        @Stack  0x01  # stack set point with address
        @Invoke 0x04
        
        @Invoke 0x80 # call kernel
1. Basic concepts
Variable

Переменная - это контейнер, который содержит значение. Думайте об этом как о коробке с надписью, в которую вы можете складывать вещи. Когда вы создаете переменную, вы даете ей имя и присваиваете ей значение.

Stack

Стек - это фундаментальная структура данных, используемая в низкоуровневом программировании. Это область памяти, которая используется для временного хранения данных. Основными операциями, которые могут быть выполнены со стеком, являются push и pop. Операция push добавляет новый элемент в верхнюю часть стека, в то время как операция pop удаляет верхний элемент из стека.

Registers

Регистры - это небольшой набор специальных ячеек памяти, к которым процессор имеет прямой доступ. Они используются для хранения данных для быстрого доступа и часто используются для получения промежуточных значений в вычислениях. Как правило, доступ к ним осуществляется быстрее, чем к памяти, что делает их идеальными для часто используемых данных.

2.Variable

В AsmX вы можете создать переменную, используя оператор '@set'. Первый аргумент - это имя переменной, второй аргумент - это тип переменной, а третий аргумент - это значение. 


        @set age Int 16;
        @set name String "E5war5IT"; 
        @set isAge Bool true;
        @set rating Float 5.0;

Здесь мы создали четыре переменные: 'возраст', 'имя', 'isAage' и 'рейтинг'. Первым аргументом является имя переменной, за которым следует ее тип (Int, String, Bool или Float) и, наконец, ее начальное значение.
